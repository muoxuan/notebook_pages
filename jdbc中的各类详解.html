jdbc中的各类详解

jdbc中的各类详解

DriverManager 驱动管理对象

位于java.sql.DriverManager，用于管理一组JDBC驱动程序的基本服务。

API文档中的解释为：

作为其初始化的一部分， DriverManager类将尝试加载在“jdbc.drivers”系统属性中引用的驱动程序类。 这允许用户自定义应用程序使用的JDBC驱动程序。

主要有两个作用：注册驱动、获取数据库连接。

注册驱动

public static void registerDriver(Driver driver) 

throws SQLException注册与给定的驱动程序DriverManager 。 新加载的驱动程序类应该调用方法registerDriver使其自己已知的DriverManager

在代码中的表示为： 

Class.forName("com.mysql.cj.jdbc.Driver");

通过反射技术将jar包中的Driver类加载进内存中。在Driver源码中有这样的静态代码块： 

static {
        try {
            DriverManager.registerDriver(new Driver());
        } catch (SQLException var1) {
            throw new RuntimeException("Can't register driver!");
        }
}

即当该类被加载进内存后，自动执行DriverManager.registerDriver(new Driver());注册驱动。

在mysql5之后，可以省略注册驱动的步骤。

注册驱动

public static Connection getConnection(String url,String user，String password) 

throws SQLException尝试建立与给定数据库URL的连接。 DriverManager尝试从一组已注册的JDBC驱动程序中选择适当的驱动程序。

url格式：jdbc:mysql://主机地址:端口号/数据库名称?serverTimezone=UTC

当连接的时本机的mysql服务器时，主机名称:端口号可以省略。即jdbc:mysql:///数据库名称?serverTimezone=UTC

Connection 数据库连接对象

Interface Connection位于java.sql包下。

API文档中解释为：

与特定数据库的连接（会话）。 执行SQL语句并在连接的上下文中返回结果。

获取执行sql的对象

Statement createStatement() 

throws SQLException创建一个Statement对象，用于将SQL语句发送到数据库。 没有参数的SQL语句通常使用Statement对象执行。 如果相同的SQL语句执行了很多次，那么使用PreparedStatement对象可能会更有效。

PreparedStatement prepareStatement(String sql) 

throws SQLException创建一个PreparedStatement对象，用于将参数化的SQL语句发送到数据库。

开启事务

void setAutoCommit(boolean autoCommit) 

throws SQLException将此连接的自动提交模式设置为给定状态。 如果连接处于自动提交模式，则其所有SQL语句将作为单个事务执行并提交。 否则，它的SQL语句被分组成通过调用方法commit或方法rollback 。 默认情况下，新连接处于自动提交模式。

void commit() 

throws SQLException使上次提交/回滚之后所做的所有更改都将永久性，并释放此Connection对象当前持有的任何数据库锁。 只有当自动提交模式被禁用时，才应该使用此方法。

void rollback() 

throws SQLException撤销在当前事务中所做的所有更改，并释放此Connection对象当前持有的任何数据库锁。 只有当自动提交模式被禁用时，才应该使用此方法。

Statement 执行sql的对象

Interface Statement 位于java.sql包下。

boolean execute(String sql) 

执行给定的SQL语句，这可能会返回多个结果。

int executeUpdate(String sql) 

执行给定的SQL语句，这可能是 INSERT ， UPDATE ，或 DELETE语句，或者不返回任何内容，如SQL DDL语句的SQL语句。

ResultSet executeQuery(String sql) 

执行给定的SQL语句，该语句返回单个 ResultSet对象。

executeUpdate方法：返回值是受影响的行数。

executeQuery方法： 返回值是结果集对象ResultSet。

ResultSet 结果集对象

Interface ResultSet 位于java.sql 包下。

API文档中的解释为：

表示数据库结果集的数据表，通常通过执行查询数据库的语句生成。

常用方法：

getNext() 

获取下一列的数据。

getXxx() 

获取Xxx类型的数据。

getXxx(int ) 

获取当前行中指定列的对象。

getXxx(String) 

获取当前行中指定列名的对象。

JDBC工具类 JDBCUtils

自己写！！！

当获取连接对象时，不想传递参数。

使用配置文件。

写一个jdbc.properties文件，其中记录下信息：

url=....
user=....
password=....
driver=....

使用properties集合类获取数据。

获取src目录下的文件可以通过使用类加载器来获取。 

ClassLoader classLoader = UseJdbc.class.getClassLoader();
//获取类加载器
Enumeration<URL> url= classLoader.getResources("jdbc.properties");
//获取配置文件的URL
URL u = url.nextElement();
String path = u.getPath();
//将URL转换为字符串类型的路径
Properties p = new Properties();
p.load(new FileReader(path));
//将配置文件加载进内存
String driver = p.getProperty("driver");
String ur = p.getProperty("url");
String user = p.getProperty("user");
String password = p.getProperty("password");
//获取数据
System.out.println(driver + " " + ur + " " + user + " " + password);

PreparedStatement对象 执行SQL的对象

SQL注入问题

当sql语句拼接时，如果参数包含sql关键字，可能会导致安全问题。

例如：登录系统，传递进的用户名为任意值，密码为a' or 'a' = 'a

此时，进行的sql语句拼接即为： 

SELECT * FROM UESR WHERE NAME = '任意值' AND PASSWORD = 'a' OR 'a' = 'a';

其中，'a' = 'a'为true，同时，前面有关键字OR，则无论数据库中是否包含这个词条，都会返回true，便查询到所有的USER数据。

为了避免SQL注入问题的发生，我们使用预编译SQL语句，即PreparedStatemen对象。

使用方法

和Statement对象使用方法相似，首先定义SQL语句的字符串。

不同点是：将想要传递进的参数使用'?'来作为占位符。

如上面的SQL语句，如果想使用PreparedStatement 语句进行查询，应该修改为： 

SELECT * FROM UESR WHERE NAME = ? AND PASSWORD = ?;

然后使用数据库连接对象connection中的prepareStatement()方法获得PreparedStatement对象。

该方法需要传递进去SQL语句。

然后使用PreparedStatement对象中的setXxx()方法对占位符赋值。Xxx指的是被赋值的参数的数据类型。

setXxx(int x, Xxx y);

x为占位符所在的位置编号，以1为起点。y为复制进去的参数。

然后使用PreparedStatement对象中的方法executeQuery或executeUpdate()方法执行SQL语句。这两个方法不需要参数。

例如： 

Class.forName(driver);
// 注册驱动
Connection conn = DriverManager.getConnection(ur,user,password);
// 获取数据库连接对象
String sql = "SELECT * FROM test1 WHERE ID = ?";
// 定义sql语句
PreparedStatement stmt = conn.prepareStatement(sql);
// 获取sql执行对象
stmt.setInt(1,id);
ResultSet r  = stmt.executeQuery();0
// 执行sql语句
while(r.next()){
    System.out.print(r.getInt(1) + " ");
    System.out.print(r.getString(2)+ " ");
    System.out.println(r.getInt(3));
}
stmt.close();
conn.close();
//释放资源

JDBC事务管理

void setAutoCommit(boolean autoCommit) 

throws SQLException将此连接的自动提交模式设置为给定状态。 如果连接处于自动提交模式，则其所有SQL语句将作为单个事务执行并提交。 否则，它的SQL语句被分组成通过调用方法commit或方法rollback 。 默认情况下，新连接处于自动提交模式。

void commit() 

throws SQLException使上次提交/回滚之后所做的所有更改都将永久性，并释放此Connection对象当前持有的任何数据库锁。 只有当自动提交模式被禁用时，才应该使用此方法。

void rollback() 

throws SQLException撤销在当前事务中所做的所有更改，并释放此Connection对象当前持有的任何数据库锁。 只有当自动提交模式被禁用时，才应该使用此方法。

当发生异常或者是数据不正确时，都进行回滚操作。

发生异常使用try...catch语句，在catch中roolback。
