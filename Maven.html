Maven

Maven

Maven是一个项目管理工具，包含有一个项目对象模型，(POM:Project Object Model)，一组标准集合()，一个项目生命周期(Project Lifecycle)，一个依赖管理系统(Dependency Management System)和运行定义在生命周期阶段(phase)中插件(plugin)目标(goal)的逻辑。

Maven的作用：

  1. 快捷导入jar包；

  2. 编译文件；

  3. 自动运行测试单元；

  4. 自动打包web项目。

Maven介绍

Maven依赖管理

Maven工程对jar包的管理。

Maven开发的项目中依赖的jar包不保存在web项目中，而是保存在Maven的jar包仓库中。大大的减少了磁盘空间的占用。同时实现代码可重用。

项目中只需保存jar包的坐标即可。

Maven的一键构建

项目从编译、测试、运行、打包、安装、部署这整个过程都交给Maven来完成。这个过程就叫做Maven的一键构建。

配置环境变量

配置环境变量：系统变量中添加：变量名：MAVEN_HOME 变量值为Maven路径。

Path中添加%MAVEN_HOME%\bin

仓库种类与关系

仓库分为三种：本地仓库、中央仓库、远程仓库(私服)

当使用Maven启动项目时，会默认去系统盘寻找本地仓库。

如果电脑可以联网，可以自动去中央仓库寻找和下载jar包。

我们可以创建远程仓库(私服)，如果电脑和远程仓库在同一个局域网，可以直接在远程仓库下载jar包。如果远程仓库没有需要的jar包，会根据中央仓库下载，或者也可以从本地上传。

Maven的标准目录结构

  • src
    • main
      • java(java核心代码部分)
      • resources(配置文件部分)
      • webapp(页面资源，css、js、图片等等)
    • test
      • java(测试代码部分)
      • resources(测试配置文件部分)

Maven常用命令

通过cmd进入到项目目录下。

mvn compile：编译java文件，将.class文件放在target目录下。

mvn clean：删除target目录，即删除掉编译出的.class文件。

mvn test：测试，将test目录下的java代码和main目录下的java代码编译，同样保存在targer文件夹下。

mvn package：打包，将java核心代码和测试代码都编译，然后将项目打包为war包。

mvn install：把项目打包后，安装在本地仓库中。

mvn deploy：将项目发布

清理生命周期

默认生命周期

compile --> test --> package --> install --> deploy

执行后边的操作，一定会执行前面的操作。

使用maven骨架创建maven的java工程

创建时选择maven下的quickstart。

添加Groupid、ArtifactId与version。

选择maven目录与setting.xml目录与本地仓库位置。

添加项目名称与项目位置。

在main文件夹下创建resources文件夹，用来存放配置文件。

右键新创建好的文件夹，选择Mark Directory as下的sources Root。作为资源根目录。

不使用骨架创建maven的java工程

创建java工程时，可以更少的创建maven的目录结构。

使用骨架创建maven的web工程

创建时选择webapp骨架。

自己添加java文件夹与resources文件夹

导入jar包

在pom.xml配置文件中添加dependencies标签。在这个便签内部添加dependency标签。在这个便签下面添加groupId标签、artifactId标签与cersion便签。

可以在https://mvnrepository.com/[maven中央仓库](https://mvnrepository.com/)处找到需要的jar包，其内部包含有坐标。复制粘贴。

运行maven的生命周期插件，选择右边的Maven Projecties标签下，选择要执行的操作，双击即可。

在同样的位置有一个小m，点击可以添加命令。

jar包冲突

为jar包添加作用域，即添加标签scope，内容为provided。表示只在编译的过程中使用这个jar包。

内容为test：表示只在测试时使用这个jar包。
