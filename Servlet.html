Servlet

Servlet

Servlet:server applet 运行在服务器的小程序。

Servlet是一个接口，定义了浏览器访问java类(TomCat调用的类)的规则。

简单使用

1、创建JavaEE项目。

2、在项目中创建Servlet实现类。并实现其方法。

3、修改配置文件web.xml，在根标签下加入

<servlet>
    <servlet-name>名称</servlet-name>
    <servlet-class>类的全路径</servlet-class>
</servlet>
<servlet-mapping>
    <servlet-name>名称</servlet-name>
    <url-pattern>/地址</url-pattern>
</servlet-mapping>

执行原理：

Tomcat开启后，当用户访问该地址。则会根据配置文件中对应的\标签的子标签\中的类名称，来寻找与之对应的\标签。获取类的全路径。找到该类之后，将其加载进内存。并创建该类的对象。调用其service()方法。

Servlet生命周期方法

Servlet生命周期：

被创建

执行public void init(ServletConfig servletConfig) throws ServletException {}方法。默认是当这个类的粒径被访问的时候才创建这个类。该方法被调用。可以通过修改配置文件的方式，修改类被创建的情况。即：在\标签下创建\标签。

当该标签的值为负数时，则是在第一次被访问时，创建。

当该标签时正数时，则是在服务器启动时被创建。

init方法只能被执行一次。说明在内存中只能存在一个Servlet对象。当多个用户同时访问这个页面时，可能会产生一些安全问题。

解决方法：

尽量不创建成员变量，只创建局部变量。

如果需要成员变量，不对成员变量进行修改。

被执行

每当页面被访问的时候，都会执行public void service(ServletRequest servletRequest, ServletResponse servletResponse) throws ServletException, IOException {}方法

被销毁

当服务器关闭时，执行public void destroy() {}方法。仅当服务器正常关闭。

该方法在Servlet被销毁之前执行，一般用于释放资源。

Servlet3.0注解配置

Servlet3.0及以上版本，可以不使用配置文件的方式来配置Servlet。

方法：使用@WebServlet注解

@WebServlet注解： 

//
// Source code recreated from a .class file by IntelliJ IDEA
// (powered by Fernflower decompiler)
//

package javax.servlet.annotation;

import java.lang.annotation.Documented;
import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;

@Target({ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface WebServlet {
    String name() default "";

    String[] value() default {};

    String[] urlPatterns() default {};

    int loadOnStartup() default -1;

    WebInitParam[] initParams() default {};

    boolean asyncSupported() default false;

    String smallIcon() default "";

    String largeIcon() default "";

    String description() default "";

    String displayName() default "";
}

通过修改该注解中的数据，可以做到配置Servlet。

默认无参配置的即是资源目录。

Servlet体系结构

Servlet接口 --> GenericServlet(Servlet实现类) --> HttpServlet(GenericServlet子类)

GenericServlet

GenericServlet类实现了Servlet接口中除了service()方法的其他默认空实现，service()方法实现为抽象方法。当我们创建一个继承GenericServlet()的类时，只需要重写service()方法即可。

HttpHttpServlet

HttpServlet继承于GenericServlet类，HttpServlet类重写了service()方法结构如下： 

public void service(ServletRequest servletRequest, ServletResponse servletResponse) throws ServletException, IOException {
         String method = req.getMethod();
          if (method.equals("GET")) {
              get方式的获取数据的方法；
              doGet();
          }
          else if (method.equals("POST")) {
              post方式的获取数据的方法；
              doPost();
          }
    }

当我们创建一个类继承于HttpServlet类的时候，直接重写doGet()和doPost()方法即可。

HTTP请求协议

Hyper Text Transfer Protocol 超文本传输协议。

定义了客户端与服务器端传输时发送数据的格式。

特点：

1、是基于TCP/IP协议的高级协议。

2、默认端口号：80.

3、基于请求/响应模型：一次请求对应一次响应。

4、无状态的：每次请求都是独立的、不能交互数据。

HTTP请求数据

  1. 请求行

    1. 请求的格式：请求方式 请求url 请求协议/版本号

    2. http协议有7种请求方式，常用的时以下两种：

      1. GET方式

        1. 参数在请求行中，url后
        2. 请求的url长度有限
        3. 不安全

      2. POST方式

        1. 参数在请求体中
        2. 请求的url长度无限制
        3. 相对安全

  2. 请求头

    1. 格式：请求头名称:请求头值 例如： 

      Accept: text/html, application/xhtml+xml, application/xml; q=0.9, */*; q=0.8
Accept-Encoding: gzip, deflate
Accept-Language: en-US, en; q=0.8, zh-Hans-CN; q=0.5, zh-Hans; q=0.3
Cache-Control: max-age=0
Connection: Keep-Alive
Cookie: JSESSIONID=8403E05D3EB4F2CC26A12E269B71E813
Host: localhost
Upgrade-Insecure-Requests: 1
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/70.0.3538.102 Safari/537.36 Edge/18.18362

    2. 常见的请求头：

      1. User-Agent:浏览器的版本信息(为了针对不同版本浏览器做出不同动作，解决兼容问题)

      2. Referer:请求此条消息的来源 目的：

        1. 防盗链

        2. 统计信息

  3. 请求空行 就是一个空行，用来分隔请求头和请求体。

  1. 请求体 封装了POST请求信息的请求参数
