Set集合

HashSet集合

API文档解释如下：

此类实现 Set 接口，由哈希表（实际上是一个 HashMap 实例）支持。它不保证 set 的迭代顺序；特别是它不保证该顺序恒久不变。此类允许使用 null 元素。

主要方法

boolean add(E e) 

      如果此 set 中尚未包含指定元素，则添加指定元素。

void clear() 

      从此 set 中移除所有元素。

Object clone() 

      返回此 HashSet 实例的浅表副本：并没有复制这些元素本身。

boolean contains(Object o) 

      如果此 set 包含指定元素，则返回 true。

boolean isEmpty() 

      如果此 set 不包含任何元素，则返回 true。

Iterator iterator() 

      返回对此 set 中元素进行迭代的迭代器。

boolean remove(Object o) 

      如果指定元素存在于此 set 中，则将其移除。

int size() 

      返回此 set 中的元素的数量（set 的容量）。

HashSet是无序存储的

public static void main(String[] args) {
    HashSet <String> ha = new HashSet<>();
    ha.add("sa11");
    ha.add("666");
    ha.add("777");
    ha.add("ssa");
}

HashSet底层结构

jdk1.8之前为数组 + 链表

jdk1.8之后为数组 + 链表/红黑树

数组记录哈希值，下方为一个链表。当数据添加的时候会先计算出其哈希值，然后寻找数组中是否包含这个哈希值，如果包含则在该哈希值对应的链表中寻找是否含有该元素，如果有则不添加，如果没有就添加到该链表当中。

数组长度为16

jdk1.8之后，如果链表长度超过8，则变为红黑树（加快查找速度）

HashSet存储自定义数据类型

由HashSet底层结构可得知，要想使用HashSet存储自定义数据类型，该数据类型需重写hashCode方法和equals方法。

原因：首先为了避免重复应该重新计算hashCode值，否则如果随机分配的话很有可能导致2个相同的数据的哈希值不同。就不能保证HashSet集合中无重复元素。且重写时保证相同元素的哈希值相同。重写equals方法原因与重写hashCode方法大同小异。

LinkedHashSet集合

与HashSet集合基本相同。但是LinkedHashSet集合为数组 + 链表/红黑树 + 链表。最后的链表记录有存储的顺序。保证其有序。

即LinkedHashSet集合是有序的，HashSet集合是无序的。

可变参数

格式： method (Object... obj)

原理为将传递进来的参数转换为数组格式，即创建一个名为ojb，数据类型为Object 的数组。

具体操作按照数组的操作即可。

注意事项：

参数列表中只能使用一次可变数组，且必须位于最后一项

如果想多次使用，则可以定义 Object... obj。
