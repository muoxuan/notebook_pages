redis

redis

概念：redis是一款高性能的NOSQL系列的非关系型数据库。

安装：

windows下安装：

下载地址：https://github.com/MSOpenTech/redis/releases。

Linux下安装

下载地址：http://redis.io/download，下载最新稳定版本。

redis的数据结构

redis存储的是key，value格式的数据，其中value有以下5中数据类型。

  1. 字符串类型 string

  2. 哈希类型 hash：map格式

  3. 列表类型 List：Linklist

  4. 集合类型 set：

  5. 有序集合类型 sortedset

redis的命令操作

  1. value是string字符串类型：

    1. 存储：set key value

    2. 获取：get key

    3. 删除：del key

  2. value是hash哈希类型：

    1. 存储：hset key field value

    2. 获取：hget key field

    3. 删除：hdel key field

    4. 获取所有的field和value：hgetall key

  3. value是list列表类型（可以将一个元素添加到列表的头部（左边）或者是尾部（右边））：

    1. 将元素添加到列表的左边：lpush key value

    2. 将元素加入到列表的右边：rpush key value

    3. 范围获取：lrange key start end （设置范围是0 -1即是获取所有元素）

    4. 从列表最左边删除元素，并返回：lpop key

    5. 从列表最右边删除元素，并返回：rpop key

  4. value是set集合类型（不允许重复）：

    1. 存储：sadd key vaule

    2. 获取set集合中所有元素：smembers key

    3. 删除：srem key value

  5. value是sortset集合：

    1. 存储的时候不仅仅存储值，还存储了一个分数表示大小。获取时，根据分数大小排序。

    2. 存储：zadd key score value（根据score进行排序）

    3. 获取指定范围元素： zrange key start end

    4. 删除：zrem key value

  6. redis的通用命令：

    1. 获取所有的key值：keys *

    2. 获取一个key值所对应的value的类型：type key

    3. 删除指定的key value：del key

redis持久化

redis是一个内存数据库，当服务器关闭或重启时，数据会丢失。我们可以把redis内存中的数据持久化的存储在硬盘中。

redis的持久化机制

RDB

RDB是默认方式，在一定间隔时间中，检测key的变化情况。然后持久化数据。不保证所有数据都不丢失，性能影响较小。

可以通过修改配置文件redis.windows.conf的方式修改配置。

save time times

当time时间内(单位是秒) 有 times 个keys发生改变就持久化一次。

启动方式：使用命令行：cd到文件夹下，redis的exe文件 redis的配置文件。

AOF

日志记录的方式。可以记录每一条记录的操作。每一次记录的操作后，持久化数据。保证所有数据不丢失，对性能影响较大。

修改配置文件：找到appendonly no，修改成appendonly yes即开启AOF

appendsync always：每一次操作都执行持久化。

appendsync everysec：每秒执行一次持久化。

appendsync no：不进行持久化。

Jedis

Jedis是一款java操作redis数据库的工具。类似于JDBC。

使用步骤：

  1. 下载Jedis的jar包

  2. 获取连接

    • Jedis jedis = new Jedis(host port);参数为指定地址和端口号。

  3. 操作

  4. 关闭连接

Jedis操作redis中的各种数据结构

Jedis.xxx();

xxx为各种指令。

特殊方法：

jedis.setex(key, seconds, value)将键值对存入，在seconds秒后删除。

当获取hash类型的所有数据时，返回类型是一个map集合。泛型是String，String

当获取lsit类型的范围数据时，返回类型是一个list集合。泛型是String

当获取set类型的所有数据时，返回类型是一个set集合。泛型是String

即：redis存储的数据都是String类型的。

Jedis连接池

Jedis自带连接池JedisPool。

使用方法：

  1. 创建配置对象JedisPoolConfig

  2. 创建Jedis连接池对象

  3. 调用方法getResorce方法获取Jedis连接。
