response

response

Http响应消息

格式：

响应行

响应头

响应空行

响应体

响应行

组成

协议/版本号 响应状态码 状态码描述

响应状态码

服务器告诉客户端浏览器本次请求和响应的一个状态标记。

由3位数字组成。

响应码分类：

1xx：服务器收到浏览器消息，但是接受没有完成。等待一段时间之后，服务器向浏览器发送该状态码。

2xx：成功发送响应。通常为200

3xx：重定向。代表响应码：302(重定向) 304(访问缓存)

4xx：客户端错误。代表响应码：404(请求的路径没有对应的资源)，405(请求方式没有对应的doxxx的方法)

5xx：服务器错误。

响应头

与请求头相似，是一些键值对。

常见的响应头：

  1. Content-Type： 服务器告诉客户端本次响应的数据格式以及编码格式。

  2. Content-disposition：服务器告诉客户端以什么格式打开数据。常见的有：

    1. in-line：默认值，直接在页面中打开。

    2. attachment;filename = xxx附件形式打开响应体(文件下载)

响应空行

与请求空行相似，用来分隔响应头与响应体。

响应体

具体传输的数据。

Response对象

功能：设置响应消息。

设置响应行

设置状态码： 

setStatus(int sc);

设置响应头

setHeader(String name, String value);

设置响应体

  1. 获取输出流

    1. 字符输出流 PrintWriter getWriter();

    2. 字节输出流 ServletOutputStream getOutputStream();

  2. 使用输出流将数据输出到浏览器中

重定向

服务器中有两个资源A和B，当浏览器访问A资源时，服务器告诉浏览器需要访问B资源。浏览器变去访问B资源。这个过程称为重定向。

步骤：

  1. 告诉浏览器需要重定向，设置状态码为302；

  2. 告诉浏览器重定向的路径，即响应头设置location:B资源的路径。

例如：

response.setStatus(302);
    response.setHeader("location","/httptest_war_exploded/test");

路径必须是完整的虚拟路径。

java中提供了简便写法：response.sendRedirect(路径名);

response.sendRedirect("/httptest_war_exploded/test");

重定向特点

有以下三个特点，这些特点与请求转发正好相反。

  1. 重定向后地址栏的地址发生改变。

  2. 重定向可以访问其他站点(服务器)的页面。

  3. 重定向是两次请求。

response字符输出流乱码问题

由于服务器和浏览器设置的字符集不同，中文字符会产生乱码。

解决方法：

在响应头处设置content-type属性。 

response.setHeader("content-type","text/html;charset=utf-8");

这条语句会同时设置服务器的字符集。

response对象中还有简便写法： 

response.setContentType("text/html;charset=utf-8");

二者可以起到相同的作用。

ServletContext对象

代表整个web应用，可以和程序的容器(服务器)进行通讯。

获取方法

有两种方法：

request.getServletContext();
this.getServletContext();

用途

获取mimeType

mimeType是一种描述文件格式的字符串。例如：text/html。通常语法为：大类型/小类型。

servletContext对象中的getMimeType(String s)方法可以将传进去的文件名根据其扩展类型返回其mimeType的字符串。

例如： 

String mimeType = servletContext.getMimeType("a.txt");
System.out.println(mimeType);//text/plain
域对象

因为servletContext代表整个应用，所以他的作用域为整个web项目。

也就是说，多个不同用户/java类等等获取/设置对象时，获取/设置的对象为同一个对象。

使用方法： 

servletContext.setAttribute("6","666");
获取文件真实路径

当项目部署在远程服务器时，为了避免因为资源位置发生改变而导致代码出现错误。可以使用servletContext对象中的getRealPath(String s)来获取真实路径。其中s为该文件的虚拟路径。

例如： 

String realPath = servletContext.getRealPath("/WEB-INF/web.xml");
System.out.println(realPath);//E:\JavaProject\httptest\out\artifacts\httptest_war_exploded\WEB-INF\web.xml
