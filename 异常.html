异常

异常

异常的概念

异常Exception，继承于Throwable类。

Throwable类有以下两个子类：Exception、Error。

Error：错误，发生了无法解决的问题，只能依靠修改源代码来避免。

Exception 编译时发生异常，即可检查异常。可检查异常必须在代码中显示捕获处理。

RuntimeException Exception的子类。为不可检查异常。多为逻辑错误，具体需要判断是否需要捕获处理异常。

异常产生的过程

例如代码

package domo01;
public class Main {
    public static void main(String[] args) {
        int a [] = {1,2,3};
        print(a,3);
    }
    public static void print(int [] a, int t){
        System.out.println(a[t]);
    }
}

抛出异常： 

Exception in thread "main" java.lang.ArrayIndexOutOfBoundsException: Index 3 out of bounds for length 3
    at domo01.Main.print(Main.java:8)
    at domo01.Main.main(Main.java:5)

首先，访问了数组a的3索引，但是a中并没有3索引。JVM便会检测出其中的异常。

于是，JVM创建了一个异常对象new ArrayIndexOutOfBoundsException(3), 其中包含有异常发生的内容、原因、地点。

然后JVM在方法print中没有找到异常的处理逻辑（try...catch）。于是JVM将这个异常抛出给调用这个方法的Main方法。

Main方法中同样没有异常处理的逻辑，所以Main方法会把这个异常抛出给JVM。

JVM接收到这个异常，首先将这个异常以红色字体打印在Java控制台上，然后终止程序的运行（中断处理）。

异常处理

throw关键字

可以使用throw关键字抛出指定的异常。

格式： throw new xxxException("....")；

...代表对异常内容的说明。

要求： xxxException必须是Exception或是其子类。 抛出throw异常就必须处理这个异常。（分为2种情况）

当抛出的是RuntimeException或其子类时，可以不用处理，交给JVM处理；

当抛出的事编译异常时，必须处理要么throws，要么try...catch。

合法性校验

当方法传递进参数是，必须对参数进行合法性校验：

例如： 

public static void print(int [] a, int t){
        if(a == null){
            throw new NullPointerException("数组为空");
        }
        if(t < 0 || t >= a.length){
            throw new ArrayIndexOutOfBoundsException("索引超出数组索引的范围");
        }
        System.out.println(a[t]);
}

当发生异常时： 

public static void main(String[] args) {
        int a [] = {1,2,3};
        print(a,3);
    }

Exception in thread "main" java.lang.ArrayIndexOutOfBoundsException: 索引超出数组索引的范围
    at domo01.Main.print(Main.java:12)
    at domo01.Main.main(Main.java:5)

public static void main(String[] args) {
        int a [] = null;
        print(a,1);
}

Exception in thread "main" java.lang.NullPointerException: 数组为空
    at domo01.Main.print(Main.java:9)
    at domo01.Main.main(Main.java:5)

Objects.requireNoNull方法

源码： 

public static <T> T requireNonNull(T obj, String message) {
        if (obj == null)
            throw new NullPointerException(message);
        return obj;
}

简化参数为空时抛出异常的操作

例如： 

public static void main(String[] args) {
    int a [] = null;
    print(a,1);
}
public static void print(int [] a, int t){
    Objects.requireNonNull(a,"数组为空");
    if(t < 0 || t >= a.length){
        throw new ArrayIndexOutOfBoundsException("索引超出数组索引的范围");
    }
    System.out.println(a[t]);
}

抛出异常： 

Exception in thread "main" java.lang.NullPointerException: 数组为空
    at java.base/java.util.Objects.requireNonNull(Objects.java:246)
    at domo01.Main.print(Main.java:11)
    at domo01.Main.main(Main.java:8)

throws关键字

抛出异常给调用该方法的方法。

接收到的方法必须对这个异常进行处理，包括再抛出

main方法则抛出给JVM

例如


public class Main {
    public static void main(String[] args) throws Exception {
        int [] a = new int [2];
        print(a,2);
    }
    public static void print(int [] a, int t) throws Exception {
        if(t < 0 || t >= a.length){
            throw new Exception("索引超出数组索引的范围");
        }
        System.out.println(a[t]);
    }
}

注意点：

1、throws的异常可以是throw异常的父类

2、Exception的子类异常必须被抛出（即编译异常），RuntimeException不是必须抛出（即运行期异常），可以直接交给JVM处理

try...catch...finally

另一种异常的处理方式

格式：

try｛  

｝catch（异常类名 对象名）｛  
    处理异常的代码...
｝finally｛
    代码段...
｝

try中是可能会发生异常的代码

catch中是如果发生了异常那么进行处理的代码

finally是无论是否发生异常，都会执行的代码(可以没有)

Throwable中3个异常处理的方法

1、getMessage方法

返回值类型：String，即该异常的文本，即新声明的异常中的异常内容

public static void print(int [] a, int t)  {
        try {
            if (t < 0 || t >= a.length) {
                throw new DomoException("索引超出数组索引的范围");
            }
        } catch (DomoException e) {
            System.out.println(e.getMessage());//索引超出数组索引的范围
        }
}

2、toString方法

更详细的说明了异常

因为是重写了Object的toString方法，可以直接使用对象名 


public static void print(int [] a, int t)  {
        try {
            if (t < 0 || t >= a.length) {
                throw new DomoException("索引超出数组索引的范围");
            }
        } catch (DomoException e) {
            System.out.println(e.toString());// domo01.DomoException: 索引超出数组索引的范围
            System.out.println(e);// domo01.DomoException: 索引超出数组索引的范围
        }
}

3、printStackTrace方法

无返回值，直接将异常的内容、原因，位置打印在控制台上。是JVM所使用的的方法


public static void print(int [] a, int t)  {
        try {
            if (t < 0 || t >= a.length) {
                throw new DomoException("索引超出数组索引的范围");
            }
        } catch (DomoException e) {
            e.printStackTrace();
        }

}

控制台打印内容： 

domo01.DomoException: 索引超出数组索引的范围
    at domo01.Main.print(Main.java:13)
    at domo01.Main.main(Main.java:8)

多异常捕获

如果方法中可能包含多个异常，那么异常处理方式为try..catch方法

1、对每个异常分别进行处理:

public static void print(int [] a, int t) {
        try {
            throw new DomoException();
        } catch (DomoException e) {
            System.out.println("01");
        }
        try {
            throw new Demo01Exception();
        } catch (Demo01Exception e) {
            System.out.println("02");
        }
}

控制台打印内容：

01
02

2、多个异常一次捕获，多次处理

public static void print(int [] a, int t) {
        try {
            System.out.println(a[t]);
            throw new Demo01Exception("666");
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("555");
        }catch (Demo01Exception e){
            e.printStackTrace();
        }
}

先处理先发生的异常，当异常发生时，会中断try中的代码段。

当catch多个异常时，子类一定得写在父类上面。否则当子类异常发生时，不会根据子类异常处理方法处理，会按照父类异常处理方法处理（多态性）

3、 一次捕获一次处理

根据第二条可得出，只要捕获的时所有异常的父类就可以实现一次捕获一次处理

子父类异常

在子类重写父类方法时：

父类抛出异常，子类可以抛出和父类一样的异常，也可以抛出父类异常的子类异常，也可以不抛出异常

但是不可以父类不抛出的异常，子类去抛出这个异常（必须在子类中进行处理）

自定义异常类

自定义异常类需满足以下格式：


class DemoException extends Exception(或者是其子类){
    DemoException(){
        super();
    }
    DemoException(String s){
        super(s);
    }
}

需要构建两个构造方法，一个是无参构造，一个是有参的构造。

构造方法直接调用父类的构造方法即可

如果自定义异常类继承的是Exception 那么它就是一个编译期异常。同样，如果继承的是RuntimeException，他就是一个运行期异常。
