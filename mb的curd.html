mybatis的CRUD操作

mybatis中的CRUD操作

通过使用代理dao的方式实现CRUD操作

增加一条语句的操作

  1. 使用配置文件的方式

    1. 在Dao接口中添加执行增加操作的方法；

    2. 在mapper配置文件中添加一条对于该方法的标签

    3. 调用即可，与查询操作相似。但是：因为mybatis默认是手动提交事务，如果执行后不进行提交，则事务会自动回滚。提交方式：SqlSession.commit()方法。

    4. 具体代码部分：

      1. mapper配置文件中mapper标签下添加标签：

        <insert id="save" parameterType="com.mybatis.domin.Note">
insert into NOTE values (#{ID},#{NOTE},#{TIME},#{TITLE})
</insert>

        parameterType属性表示参数的类型，即被传递进Dao接口代理类中该方法的参数类型。Sql语句values中每个值都被#{}的结构包围。其内部为参数中属性的名称。

      2. Dao接口

        void save(Note note);

  2. 使用注解的方式配置

    1. 在Dao接口中添加执行增加的方法

    2. 在该方法的上方添加@Insert注解，参数为要被执行的Sql语句。

    3. 这个Sql语句的语法与注解配置的方法是一样的，创建代理类时，会通过使用反射技术获取到传递进的参数，从而可以直接为Sql语句中的参数进行赋值。

    4. 具体实现代码

      @Insert("insert into NOTE values(#{ID},#{NOTE},#{TIME},#{TITLE})")
void save(Note note);

插入操作后，如果id为自动增长的，同时，我们想获取到这个id，可以使用以下方法：

在插入操作的标签内添加标签


<selectKey resultType="" keyProperty="" order="" keyColumn="">
    select last_insert_id();
</selectKey>

参数说明：resultType：返回值类型，order：执行顺序(BEFORE/AFTER) keyProperty：结果应该被设置的目标属性。因为id属性为自动增长的属性。所以添加KeyProperty属性后，会自动将获取到的id赋值给传递进去的对象中的id属性。

修改、删除操作

与之前的操作类似

代码：

xml文件

<update id="change" parameterType="com.mybatis.domin.Note">
    update NOTE set NOTE = #{NOTE},TIME = #{TIME}, TITLE = #{TITLE} where ID = #{ID};
</update>
<delete id="delete" parameterType="java.lang.String">
    delete from NOTE where ID = #{id};
</delete>

java

void change(Note note);

void delete(@Param("id")String id);

注意，在使用String类型的参数时，需要在方法参数前添加进Param注解。 因为在使用#{}方式传递参数时，会自动寻找参数的getter方法。而String中没有getter方法。所以需要提阿奴家@Param注解。

查询操作

模糊查询

与之前的内容类似，只需要将模糊的字符串传递进去即可。

聚合函数

直接按照sql语句的语法进行填写xml配置文件以及Dao接口中方法的填写即可。

传递的参数深入

OGNL表达式

Object Graphic Navigation Language ：对象图导航语言

通过对象中的取值方法获取数据、写法上省略get

例如：user.username获取用户名

所以，当传递进来的参数中包含其他类的对象时，想要获取到这个对象中的属性，可以使用对象名.属性名的方法获取。

返回值深入

mysql数据库在windows环境下忽略大小写，但是在linux环境下是严格区分大小写的。

所以为了返回正确的对象(即把对应的数据封装进对象中，需要列名与属性名一一对应)。

如果不对应，应该如何解决：

  1. 在Sql语句中起别名

    例如：Select Id as userID from tablename

  2. 配置查询结果的列名和实体类属性名的对应关系

<!--  设置对应关系  -->
<resultMap id = "随便起", type = "对应实体类的全限定类名">
    <!-- 主键  -->
    <id property = "java中的属性名称" colum = "列名">
    <!--  非主键字段  -->
    <result property = "java中的属性名称" colum = "列名">
</resultMap>

<!-- 配置 -->

<select id="findAll" resultMap="随便起">
    SELECT * FROM Note
</select>

这种方式执行效率较慢，但是开发效率很高

properties配置

在主配置文件中可以添加properties标签，将数据库连接信息添加到这个标签底下。这种方式支持将配置文件放在该xml文件之外。使用properties类型文件。此时在properties中添加resource属性。值为properties文件的路径。按照类路径的格式去编写。同时将下面的配置信息改为#{}形式的参数，参数名称和properties属性名称必须一致。

url属性：按照url的格式来填写路径。

写法：协议 主机 端口 URI

windows文件系统使用file协议，即 file: ///资源路径

typeAliases标签

在使用type标签时，可以直接填写int、String等等。

原因：默认起别名

在typeAliases标签下添加typeAlias标签，修改type属性，添加进去原本的全限定类名，alias属性指定别名。指定别名后，不再区分大小写。

package标签

指定要配置别名的包，当指定后，可以指定当前包下的所有类，别名即为类名。
